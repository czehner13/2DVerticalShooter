Unity 2D Space Shooter Game


In today's Tutorial we will learn how to make a 2D Space Shooter Game in Unity with less than 100 Lines of Code. As usual everything will be explained as easy as possible so everyone can understand it.

Foreword

Our 2D Space Shooter will be inspired by the old arcade games that everyone loved when growing up.

We will make use of all kinds of different Unity features like Physics (including Rigidbodies and Colliders), Animations (Mecanim), Scripting (C#), Prefabs, Shaders and the Sprite Editor. The game art will be very easy to create with our drawing tool of choice.

Requirements

Knowledge

We will explain every single step in detail, but it's still a good idea to get a basic overview of Unity if you never used it before. Feel free to read through our easier Unity Tutorials like Unity 2D Pong Game to see what Unity is all about.

If you used Unity before then we can get started right away.

Our Space Shooter will use Unity 5.0.1f1. Newer versions should work fine as well, older versions may or may not work. The free version of Unity 5 now comes with all the engine features, which makes it the recommended version.

using System.Collections;

    void Start () {



    }



    // Update is called once per frame

    void Update () {



    }

}

We won't need the Start or the Update function so let's remove them. It's considered best practice to modify the UV Mapping in the LateUpdate function after all other calculations were finished. We will also add a public variable that let's us modify the Scroll Speed in the Inspector later on:

using UnityEngine;

using System.Collections;



public class UVScroll : MonoBehaviour {

    public Vector2 speed;



    void LateUpdate() {

        GetComponent<Renderer>().material.mainTextureOffset = speed * Time.time;

    }

}

Note: while it looks really simple, there are quite a few things happening behind the Scenes here. First of all, we used a Vector2 for our speed variable to make sure that we can modify the x (horizontal) and y (vertical) speed. We use GetComponent<Renderer>() to access the renderer component. Using GetComponent<Renderer>().material makes sure that the UV offset is not modified permanently, as it would be the case when modifying it by hand. Instead by using GetComponent<Renderer>().material Unity creates a runtime copy of the Material and deletes it after the game is stopped. Furthermore we use the mainTextureOffset property to modify the main texture's UV offset. We then use Time.time to achieve the Scrolling effect because Time.time is the elapsed time since the beginning of the game. And since this time always increases smoothly, we might as well use it for our Scrolling. Furthermore we multiply the time by our speed to slow it down or make it faster. And because our speed variable is a Vector2, it will still be one after multiplying it with Time.time. So in the end we get a new Vector2 that will be used for the UV Offset.

Ship Movement

The player should be able to control the ship by pressing the arrow keys. The movement should be physically correct, which means that the ship shouldn't be able to move through meteors, enemy ships or other things that we may encounter in space.

We will need to add a Collider to our Ship in order to make it part of the Physics world. Without a Collider our ship is still visible in the game, but it's really just that - a visual effect, just an image. Nothing would collide with it.

Let's select the ship in the Hierarchy and then click Add Component->Physics 2D->Box Collider 2D in the Inspector to add the Collider:

If our ship would be a static object like a wall then we would be done already. However since our ship is also supposed to move through the physics world, it will also need a Rigidbody.

Note: a Rigidbody is a very common component in many game & physics engines. Everything that wants to move through the game world while making use of Physics will need a Rigidbody. Usually this applies to things like the Hero, Cars, Enemies and Bullets.

We can add a Rigidbody by selecting Add Component->Physics 2D->Rigidbody 2D in the Inspector. We will use the following settings for our Rigidbody:

Note: we disabled the Gravity because there is none in a Space Shooter. We also enabled Fixed Angle to make sure that the Physics engine doesn't try to rotate the ship in case of collisions.

Alright now that our ship physics are all set up we can get to the Movement Script. All we have to do is find out if one of the Arrow Keys is pressed and if so move into the direction. Whenever possible we should try to use the Rigidbody component for movement, which pretty much allows us to use AddForce and velocity.

The velocity is the movement direction * speed. The following image shows us different movement directions:

public class Move : MonoBehaviour {

    void Start () {



    }

    }

Let's start by adding a speed variable for the ship. We will make it public so that we can modify it in the Inspector later on. We can also remove the Start and Update functions because we won't need them:

using UnityEngine;

using System.Collections;



public class Move : MonoBehaviour {

    // Ship Speed

    public float speed = 5;

}

We will however need the FixedUpdate method. The difference between Update and FixedUpdate is that Update is called about 60 times per second, depending on how many FPS we have. FixedUpdate however is called in a certain time interval. The thing to remember about this is that all Physics calculations happen in the exact same time interval, which means that we should use FixedUpdate whenever we modify any Physics stuff like our Rigidbody.

So let's create the FixedUpdate method:

using UnityEngine;

using System.Collections;



public class Move : MonoBehaviour {

    // Ship Speed

    public float speed = 5;



    void FixedUpdate() {



    }

}

Note: Start, Update and FixedUpdate are part of the functions that are automatically called by Unity. So it's important to name it exactly FixedUpdate with the same upper- and lowercase spelling.

There are several ways to check for input here. For example, we could just check if the user pressed the left or right arrow key and then do horizontal movement. Of course some users like to use the WSAD keys too, so we would also have to check for the A and D keys to do horizontal movement. Now some users might even use a gamepad, which means that we should check this too.

As usual Unity makes our lives easy by providing the GetAxis function which automatically checks all the horizontal or vertical keys and then returns a value between -1 (for left) and 1 (for right). Or 0 if nothing was pressed:

void FixedUpdate() {

    // Get Input from Arrow Keys, WSAD, Gamepads, ...

    float h = Input.GetAxisRaw("Horizontal");

    float v = Input.GetAxisRaw("Vertical");

}

Note: we used GetAxisRaw instead of GetAxis to prevent any kinds of smoothing. It just makes the ship feel more responsive.

Now we can use our h and v values to create a direction, multiply it with speed and then make it the Rigidbody's velocity:

void FixedUpdate() {

    // Get Input from Arrow Keys, WSAD, Gamepads, ...

    float h = Input.GetAxisRaw("Horizontal");

    float v = Input.GetAxisRaw("Vertical");



    // Set the Rigidbody's Velocity

    Vector2 dir = new Vector2(h, v);

    GetComponent<Rigidbody2D>().velocity = dir.normalized * speed;

}

Note: we used the normalized direction to make sure that its length is always exactly 1. This is necessary to prevent the ship from moving too fast into diagonal directions (for example when the Up and Right Arrow Keys are pressed at the same time). This problem is based on the mathematical fact that a vector like (1, 0) has the length 1, but the vector (1, 1) actually has the length 1.41. We also multiplied the direction with speed to make it 'longer'. This is what makes the ship move faster.

The movement itself is done, but we still have to set our animation state machine's Flying parameter in order for the animation to look properly. This is very easy, all we have to do is get access to the Animator component and then use its SetBool function to set our parameter to true or false depending on the vertical (y) direction:

void FixedUpdate() {

    // Get Input from Arrow Keys, WSAD, Gamepads, ...

    float h = Input.GetAxisRaw("Horizontal");

    float v = Input.GetAxisRaw("Vertical");



    // Set the Rigidbody's Velocity

    Vector2 dir = new Vector2(h, v);

    GetComponent<Rigidbody2D>().velocity = dir.normalized * speed;



    // Set Animation Parameter

    GetComponent<Animator>().SetBool("Flying", (v > 0));

}

Note: the expression (v > 0) results in a boolean value. It is true if v is greater 0 and false otherwise.

There is another thing to add here. Upon pressing the Space key we want to shoot the bullet. Or in other words, we want to load the bullet into the game. The bullet will take care of flying upwards on its own.

Here is how we can do it:

using UnityEngine;

using System.Collections;



public class FirePlayer : MonoBehaviour {

    // Bullet Prefab

    public GameObject bullet;



    // Update is called once per frame

    void Update () {

        if (Input.GetKeyDown(KeyCode.Space))

            Instantiate(bullet,

                        transform.position,

                        Quaternion.identity);

    }

}

Note: transform.position is the current position and Quaternion.identity is the default rotation.

So far our Script does exactly what it's supposed to do. But there is one last adjustment to be made here. We want to make sure that the Player's own bullet will never collide with the Player's ship:

// Update is called once per frame

void Update () {

    if (Input.GetKeyDown(KeyCode.Space)) {

        // Spawn the Bullet

        GameObject g = (GameObject) Instantiate(bullet,

                                                transform.position,

                                                Quaternion.identity);

        // Ignore Bullet<->Player collisions

        Physics2D.IgnoreCollision(g.GetComponent<Collider2D>(),

                                  transform.parent.GetComponent<Collider2D>());

    }

}

Note: using transform.parent (which references a GameObject's parent in the Hierarchy) will make sense very soon.

Now we can use Unity's InvokeRepeating function to instantiate a new bullet every few seconds:

using UnityEngine;

using System.Collections;



public class FireEnemy : MonoBehaviour {

    // The Bullet Prefab

    public GameObject bullet;



    // The firing Interval

    public float interval = 2;



    // Use this for initialization

    void Start () {

        // Call Fire every few seconds

        InvokeRepeating("Fire", interval, interval);

    }



    void Fire() {

        // Spawn the Bullet

        GameObject g = (GameObject)Instantiate(bullet,

                                               transform.position,

                                               Quaternion.identity);



        // Ignore Bullet<->Enemy Ship collisions

        Physics2D.IgnoreCollision(g.GetComponent<Collider2D>(),

                                  transform.parent.GetComponent<Collider2D>());

    }

}

Note: InvokeRepeating calls the Fire function every few seconds. The second parameter is the time until the first call and the third parameter is the repeating interval. The Fire function simply instantiates the bullet prefab at the current position with the default rotation. Afterwards we use Physics2D.IgnoreCollision to make sure that the bullet never hits it's own ship. We used transform.parent because the Bullet Spawn is the child of the enemy ship, so in order to access the enemy ship we have to use the parent property.

using UnityEngine;

using System.Collections;



public class Spawn : MonoBehaviour {

    // The Ship

    public GameObject ship;



    // The Interval

    public float interval = 1;



    // Use this for initialization

    void Start () {

        InvokeRepeating("SpawnNext", interval, interval);

    }



    void SpawnNext () {

        Instantiate(ship, transform.position, Quaternion.identity);

    }

}

Summary

In this Tutorial we created a very clean foundation for a Space Shooter. We learned all the techniques to bring our ships to life, now it's up to the reader to make the game as fun as possible.

There are all kinds of improvements that could be implemented, for example:

  • Explosions
  • More enemy ships
  • A Menu
  • A Win/Lose Screen
  • More Animations
  • A High Score
  • Ship Upgrades
  • Different Levels

And always remember: it's Unity, it's easy! Most of the things can be done without writing any code at all.
